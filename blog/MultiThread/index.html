Multithread

If multiple threads read and write on the same trunk of memory, it will have unpredictable and often destructive results.

Race Condition

A race condition occurs if the effect of multiple threads on shared data depends on the order in which threads are schedules.

Solution

Thread can access can access object only one at a time.

Java: synchronize access to shared objects - locks. Lock activated with keyword "synchronized".

Example

A full stack of cards consists of 52 cards and let four player pick cards. If we don't have locks on the methods, eventually the sum of cards in each player's hand will not be 52, because multiple players are grabing and changing the number of cards in the stack simultaneously. In contrast, if the methods have lock (are synchronized), the cards will be distributed evenly.
