Thread in Java

Some from the lecture CS301 Software Development online video

Why we need threads?

1. Speed up

2. Better class design

3. UI responsiveness

General Process of implementing threads

1. A class that implements interface Runnable

2. Runnable has one method: void run( )

3. Place thread action into run method

4. Construct object of runnable class

5. Construct thread from that object

6. Start thread

Ways to switch threads

1. yield( )

2. sleep(long mills)

States of a thread

start() --> Running

Running --> yield() --> Runnable

Running --> sleep()/wait() --> Blocked

Blocked --> notify()/notifyAll() --> Runnable

Runnable --> Running

Notes:

1. All states can directly go to terminated state

2. After the time in sleep() collapse, the thread goes to runnable state

Terminating threads

1. Natural death: dyingThread.join() //passive waiting, better while (dyingThread.isAlive()){Thread.sleep(1000)} //constantly checking

2. request for suicide: t.interrupt(); Use sleep() or wait() to sense interruptedException and do cleanup
